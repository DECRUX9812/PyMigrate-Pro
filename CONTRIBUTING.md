# Contributing to PyMigrate-Pro

Thanks for considering contributing — we appreciate it! This document explains our recommended workflow and expectations.

## How to contribute

- Fork the project.
- Create a descriptive branch: `git checkout -b feat/short-description`.
- Keep changes small and focused.
- Run tests and linters locally before submitting a PR.

## Branch naming

- feat/<short-description> — new features
- fix/<short-description> — bug fixes
- chore/<short-description> — non-functional changes (docs, ci)

## Pull request checklist

- [ ] The PR has a clear title and description
- [ ] Linting checks pass
- [ ] CI green (GitHub Actions)
